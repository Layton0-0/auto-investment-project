# 전체 하위프로젝트 QA 계획 (Shrimp Task Manager 연동)

**작성일:** 2026-03-05  
**목적:** 헤지펀드 급 퀀트트레이더 시스템에 대한 전체 하위프로젝트 포함 QA를 상세·세분화된 PLAN으로 수립하고 Shrimp Task Manager에 등록.

---

## 1. 목표

- **대상:** 모든 하위 프로젝트  
  `investment-backend`, `investment-frontend`, `investment-infra`, `investment-data-collector`, `investment-prediction-service` (참조용 `smart-portfolio-pal` 제외)
- **실행:** `scripts/run-full-qa.ps1` 파이프라인  
  Backend JUnit → API 시나리오 → Python QA → Python 단위테스트 → Frontend E2E → npm audit
- **산출:** `plans/qa/reports/YYYYMMDD-HHmm-qa-report.md` 통합 리포트
- **실패 시:** `.cursor/rules/ai-workflow-qa.md`에 따른 5단계(원인 분석) → 6단계(수정·PR) → 7단계(재테스트) 루프

---

## 2. 전제 조건

- **로컬 풀스택:** `investment-infra/docker-compose.local-full.yml` 기동  
  - Backend: 8080  
  - data-collector: 8001  
  - prediction-service: 8000  
  - Frontend 등
- **환경 변수**
  - API 시나리오: `QA_USERNAME`, `QA_PASSWORD` (미설정 시 `investment-backend/.env`의 `SUPER_ADMIN_USERNAME`, `SUPER_ADMIN_PASSWORD` 사용)
  - Python QA: `QA_DATA_COLLECTOR_URL`(기본 8001), `QA_PREDICTION_URL`(기본 8000)
- **타임아웃:** `run-full-qa.ps1` 실행 시 600000ms(10분) 이상 권장 (`.cursor/rules/local-dev-hygiene.md`)

## 3. Shrimp에 등록된 태스크 (실행 순서)

| 순서 | 태스크 이름 | 의존 | 검증 기준 |
|------|-------------|------|-----------|
| 1 | **QA-Prep: Full stack and QA environment** | 없음 | 8080, 8001, 8000 응답, API 시나리오용 인증 정보 존재 |
| 2 | **QA-Backend: JUnit tests** | QA-Prep | `investment-backend/scripts/run-tests.ps1` exit 0 |
| 3 | **QA-API: API scenario run** | QA-Backend | `scripts/run-api-qa.ps1` exit 0, 시나리오별 예상 상태·응답 키 통과 |
| 4 | **QA-Python: Python services QA** | QA-API | `scripts/run-python-qa.ps1` exit 0, 8001/8000 엔드포인트 검증 |
| 5 | **QA-PythonTests: prediction-service unit tests** | QA-Python | `investment-prediction-service` unittest discover exit 0 또는 환경 미비 시 SKIP 문서화 |
| 6 | **QA-E2E: Frontend Playwright E2E** | QA-PythonTests | `investment-frontend`에서 `npm run e2e` exit 0 |
| 7 | **QA-Security: Frontend npm audit** | QA-E2E | `npm audit --audit-level=high` exit 0 |
| 8 | **QA-Report: Integrate report and failure loop** | QA-Security | 통합 리포트 [전체]: 통과, 또는 실패 시 원인 분석·수정·재테스트 후 통과 |

## 6. 한 번에 전체 QA 실행

의존성을 순서대로 만족한 뒤, 최종적으로 한 번에 실행하려면:

```powershell
.\scripts\run-full-qa.ps1
```

리포트는 `plans/qa/reports/YYYYMMDD-HHmm-qa-report.md`에 생성된다.  
Shrimp에서는 **QA-Report: Integrate report and failure loop** 태스크에서 위 스크립트 실행 및 리포트 검토·실패 루프를 수행하면 된다.
