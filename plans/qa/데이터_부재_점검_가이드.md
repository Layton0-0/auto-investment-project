# 데이터 부재 점검 가이드 (시그널 0건·일봉 없음)

**목적**: 자동투자 현황에서 시그널 0건·파이프라인 요약 비어 있을 때 **점검 순서·원인·데이터 생성 절차**를 한 문서에서 참조한다.

## 1. 점검 순서 (시그널 0건 시)

| 순서 | 항목 | 확인 방법 |
|------|------|-----------|
| 1 | **파이프라인 기준일** | `GET /api/v1/pipeline/summary?accountNo=...` — basDt 미입력 시 **전일(어제)** 기준. 전일 팩터 계산 완료 여부 확인. |
| 2 | **준비 상태** | `GET /api/v1/ops/auto-trading-readiness` → `dailyStockRowCount`, `signalScoreRowCount` 확인. 0이면 선행 데이터·팩터 미실행. |
| 3 | **수동 트리거** | 일봉·시그널이 비어 있으면 **아래 §2 순서대로** 트리거 실행 후 readiness·pipeline/summary 재확인. |
| 4 | **원인 분석** | §3 참고 — KRX_AUTH_KEY·한투 폴백·env·스케줄 타이밍. |

---

## 2. 데이터 생성 절차 (수동 트리거 순서)

선행 의존성을 지키기 위해 **아래 순서**로 실행한다.

1. **KRX 일별 수집**  
   `POST /api/v1/trigger/krx-daily` (인증 필요)  
   → TB_DAILY_STOCK (MARKET=KR) 적재.

2. **US 일별 수집**  
   `POST /api/v1/trigger/us-daily` (인증 필요)  
   → TB_DAILY_STOCK (MARKET=US) 적재.

3. **팩터 계산**  
   `POST /api/v1/trigger/factor-calculation` (인증 필요)  
   → TB_SIGNAL_SCORE 적재.

4. **재확인**  
   `GET /api/v1/ops/auto-trading-readiness` → row 수 증가 확인.  
   `GET /api/v1/pipeline/summary?accountNo=...` → 시그널·요약 노출 확인.

**Backend 재시작이 16:00/17:00(KST) 이후인 경우**: 당일 KRX(16:00)·US(17:00) 스케줄이 이미 지나 있으므로 위 1–2 수동 트리거 후 3 실행.  
→ [13-manual-operator-tasks.md §1.8](../../investment-backend/docs/06-deployment/13-manual-operator-tasks.md) 참조.

## 3. 원인 분석 (왜 0건인가)

| 원인 | 확인 방법 | 조치 |
|------|-----------|------|
| **KRX AUTH_KEY 미설정/만료** | 로그: `KRX AUTH_KEY 미설정: 조회 스킵` WARN. `GET /actuator/health` → `dataCollectionHealthIndicator.krxAuthConfigured`. | `investment-backend/.env`에 `KRX_AUTH_KEY` 또는 `investment.data.krx.auth-key` 설정. [04-krx-api-required.md](../../investment-backend/docs/08-setup-guides/04-krx-api-required.md) 참조. |
| **한투 폴백 사용 시 API/연결** | KRX 1차 실패 후 한투 폴백 사용. 해당 계정의 App Key·Secret·연결 상태. | 13-manual-operator-tasks §1.11, 한투 API env·토큰 확인. |
| **US 수집 미설정** | 로그: `US 시장 일별 수집 스킵: collector-url·yfinance-script-path 미설정`. | `.env`: `US_COLLECTOR_URL`, `US_SYMBOLS` 또는 Backend 설정. data-collector 8001 기동. |
| **스케줄 미실행** | Backend 재시작이 16:00/17:00 이후 → 당일 KRX/US 스케줄 이미 경과. | §2 수동 트리거 1→2→3 실행. |
| **팩터 계산 미실행** | 전일 일봉은 있는데 시그널 0건. | `POST /api/v1/trigger/factor-calculation` 실행. |

## 4. B-2 검증 (수동 트리거 후 DB/readiness 확인)

- **API 검증**: 수동 트리거 1→2→3 실행 후 `GET /api/v1/ops/auto-trading-readiness` 호출해 `dailyStockRowCount`, `signalScoreRowCount` 증가 여부 확인.
- **DB 직접 확인**(선택): [plans/qa/scripts/배치_실행_이력_점검.sql](./scripts/배치_실행_이력_점검.sql) 또는 TB_DAILY_STOCK·TB_SIGNAL_SCORE row 수 조회.
- **스크립트**: 인증이 되어 있으면 `run-api-qa.ps1`로 로그인 후 curl/Invoke-RestMethod로 trigger·readiness 호출 가능. 별도 스크립트는 [scripts/verify-data-pipeline.ps1](./scripts/verify-data-pipeline.ps1) 참고(없으면 QA 팀이 동일 순서로 수동 실행).
