# 실전 배포 전 필수 확인 체크리스트

**목적**: Live 환경 배포 전 시스템 안정성·수익성 담보를 위해 반드시 점검해야 할 개발·검증 항목을 정리한다.  
**참조**: [자동투자_전체_로직_통합.md](../../자동투자_전체_로직_통합.md), [02-development-status.md](../../investment-backend/docs/09-planning/02-development-status.md). 상세 설계는 실전_배포_전_검증_체크리스트 계획서 §2·§3 참고.

## 1.1 데이터 파이프라인 및 정합성

| 점검 항목 | 핵심 확인 사항 | 현재 구현 상태(요약) | 점검 방법 |
|-----------|----------------|----------------------|-----------|
| 웹소켓 연결 안정성 | 장중 끊김 시 자동 재연결, 누락 틱/호가 REST 보완 | 재연결·구독 복원 있음; 갭 구간 보완은 "현재가"만 REST Fallback | 14-multi-account-realtime-streaming.md §2 검증 시나리오 |

| 공분산 행렬 연산 부하 | CorrelationPenalty O(N²) 지연 여부 | CorrelationPenaltyService 쌍별 상관·첫 고상관 시 스케일만; 전체 행렬 캐싱 없음 | 유니버스 크기별 배치 소요 시간 측정, 필요 시 캐싱/최적화 |

## 1.2 실행 및 체결 관리

| 점검 항목 | 핵심 확인 사항 | 현재 구현 상태(요약) | 점검 방법 |
|-----------|----------------|----------------------|-----------|
| 상태 불일치(Reconciliation) | DB 포지션 vs 브로커 잔고 불일치 감지·동기화 | 미구현. 요구사항에 Re-sync·재기동 시 KIS vs DB 검사 언급만 존재 | ReconciliationService·정기 배치·Re-sync API 설계·구현 필수 |
| 알고리즘(TWAP/VWAP) 예외 처리 | 서킷브레이커·호가창 뎁스 마름 시 미체결 잔량 처리 | AlgorithmicOrderServiceImpl·PipelineExecutor(useAlgoExecution) 존재; 서킷/뎁스 전략 문서·코드 명시 부족 | 시나리오 테스트·정책 문서화(취소 후 재진입 vs 포기) |
| 부분 체결 엣지 케이스 | 잔여 수량만 트레일링 스탑·손절 재계산 | Order.partialExecute·PARTIAL 상태 있음; ExitRule/포지션은 StrategyPosition.quantity 기준 — 부분 체결 시 포지션 분할/업데이트 흐름 검증 필요 | 단위/통합 테스트: 부분체결 → 포지션 갱신 → 청산 규칙 |

## 1.3 리스크 통제 및 거버넌스

| 점검 항목 | 핵심 확인 사항 | 현재 구현 상태(요약) | 점검 방법 |
|-----------|----------------|----------------------|-----------|
| Global Kill Switch | 비정상 폭주 주문 시 즉시 차단 | TradingHaltService·PreTradeComplianceEngine·GET/PUT /api/v1/system/kill-switch(ADMIN) 구현됨 | 장애 시나리오 테스트, (선택) 외부 킬스위치 |
| 레짐 민감도(2σ) | 휩쏘 방지 파라미터 튜닝 | CapitalDrawdownConstraintService 등; 2σ 기준값·백테스트 튜닝 문서화 여부 확인 | 백테스트로 파라미터 검증·문서 반영 |
| PnL 계산 정확성 | DailyPnl vs 계좌 Net PnL 오차 0.1% 이내 | DailyPnlService는 시초/현재 평가액 차이 기반 %; FrictionCost 미차감 — Net PnL과 직접 비교 시 오차 가능 | FrictionCost 반영 여부 결정 후 검증 또는 별도 Net PnL 검증 API |
