# 알고리즘 트레이딩 전략 설계 가이드 (Quant Strategy Design)

**작성일**: 2026-03-13  
**목적**: 알파 생성·통계적 엣지·시장 미세구조·팩터 투자·모멘텀/평균회귀/돌파 전략을 포함한 **수익성 있는 알고리즘 트레이딩 전략** 설계 시 공통 템플릿 및 원칙.  
**참조**: [00-strategy-registry.md](../../investment-backend/docs/02-architecture/00-strategy-registry.md), [quant-and-backtest.md](../../.cursor/rules/quant-and-backtest.md)

## 1. 설계 원칙 (Design Principles)

| 원칙 | 설명 |
|------|------|
| **가설 기반** | 데이터 마이닝이 아닌, **시장 비효율/행동/구조**에 대한 명시적 가설에서 출발한다. |
| **단순성** | 설명하기 어려운 전략은 프로덕션 부적합. **단순하고 재현 가능**한 로직을 선호한다. |
| **로버스트성** | 레짐 변화·꼬리 사건·데이터 결함에서도 생존 가능하도록 설계한다. |
| **커브 피팅 회피** | In-sample 최적화에 의존하지 않는다. Walk-forward·OOS·스트레스 테스트 필수. |
| **리스크 우선** | 수익보다 **자본 보존·MDD·볼atility 타게팅**을 먼저 설계한다. |

## 2. 전략 템플릿 (필수 포함 항목)

모든 신규/개선 전략은 아래 항목을 **반드시** 정의한다.

### 2.1 엣지 정의 (Expected Edge)

- **착취 대상 비효율**: (예: 단기 모멘텀 지연, 평균회귀 과매도/과매수, 변동성 돌파 시 추세 지속)
- **시간 지평**: 단기(일~주) / 중기(월~분기) / 장기(분기~연)
- **엣지 원천**: 통계적(히스토리)·행동적(과신/손실회피)·구조적(유동성·수수료·시장 미세구조)

### 2.2 진입 조건 (Entry Conditions)

- **필수 시그널**: 정량적 조건 (예: RSI < 35, 20일 고점 돌파, 듀얼 모멘텀 상위 10%)
- **필터**: 유동성·변동성·레짐(VIX·추세) 등으로 false signal 제거
- **PIT 준수**: 해당 `bas_dt` 종료 시점까지 가용 데이터만 사용 (look-ahead 금지)

### 2.3 청산 조건 (Exit Conditions)

- **익절**: 목표 수익률 도달, Trailing Stop(고점 대비 -3% 등), RSI 과매수(≥70) 등
- **손절**: 고정 %(예: -5% / -10%), ATR 배수 손절, 전저점 이탈
- **시간 컷**: N일 내 목표 미도달 시 전량 청산 (단기 전략)

### 2.4 포지션 사이징 (Position Sizing)

- **리스크 기반**: 1회 매매당 총자산 N%(예: 1%) 또는 ATR 기반 Stop 거리로 수량 산출
- **상한**: 종목당 비중 캡(예: 5%), 변동성 역가중(1/σ) 적용
- **Half-Kelly**: 승률·손익비 검증 후 50% 켈리만 적용 (초기에는 고정 비율 권장)

### 2.5 리스크 관리 (Risk Management)

- **포트폴리오 수준**: MDD 한도(예: -15%), 일일 손실 한도, 레짐 게이트(VIX > 30 시 신규 진입 축소)
- **전략 수준**: 최대 노출·레버리지 상한, Kill Switch
- **모니터링**: PnL·노출·주문 상태·리스크 메트릭 실시간 추적

## 3. 전략 유형별 설계 요약

### 3.1 모멘텀 (Momentum)

| 항목 | 내용 |
|------|------|
| **엣지** | 수익률 추세의 단기 지속(underreaction to news). 기간별 수익률 가중합으로 강한 상대 모멘텀 선별. |
| **진입** | 기간별 수익률(예: 21/63/126일) 가중 스코어가 유니버스 상위 N%; 시장(벤치) 대비 우위. 필터: 유동성, RSI/MACD 확인. |
| **청산** | 추세 훼손(모멘텀 순위 하락), -10% 손절, Time-Cut(단기), 또는 리밸런싱 주기(중기). |
| **포지션** | 변동성 역가중(1/σ); 리스크 기반 캡. |
| **기대** | 중기 보유 시 위험 조정 수익(Sharpe) 개선; 레짐 전환기 DD 확대 가능 → 레짐 필터 권장. |

**프로젝트 반영**: 듀얼 모멘텀(US), 단기 RSI>60 & MACD 필터(KR) — [00-strategy-registry.md §3.2](https://github.com/...).

### 3.2 평균회귀 (Mean Reversion)

| 항목 | 내용 |
|------|------|
| **엣지** | 과매도/과매수 구간에서 가격이 평균으로 회귀. 이격도·RSI·Bollinger Band 등. |
| **진입** | RSI(14)<40 (또는 30), 이격도 85 이하, P/B 0.8~0.9(저평가). 필터: 유동성, 펀더멘털 훼손 아님. |
| **청산** | RSI≥70 익절, -5% 고정 손절, 전저점 이탈 손절. |
| **포지션** | 고정 비율 또는 ATR 손절 기반; 단기이므로 1종목당 비중 제한. |
| **기대** | 변동성 구간에서 승률 보완; 강한 추세장에서는 손실 확대 → 추세 필터(예: 200일선)로 구간 제한. |

**프로젝트 반영**: 한국 역발상(Case B) RSI<40·P/B 0.8 이하, 이격도 85 이하 분할 매수 — [00-strategy-registry.md §3.2](https://github.com/...).

### 3.3 돌파 (Breakout)

| 항목 | 내용 |
|------|------|
| **엣지** | 범위(레인지) 이탈 시 추세 가속. 변동성 돌파(Open + Range×k), N일 고점/저점 돌파. |
| **진입** | 목표가 = 시가 + (전일 High−Low)×k (k=0.3~0.7, 동적 조정 가능); 당일 목표가 돌파 시 매수. 필터: 거래대금(예: 300억 이상). |
| **청산** | 당일 청산 또는 Trailing Stop(-3%), -5% 손절; Time-Cut. |
| **포지션** | 1회 리스크 % 고정, ATR 기반 손절 거리로 수량 결정. |
| **기대** | 변동성 확대일 수익; 횡보장에서 Whipsaw 많음 → 유동성·k 값 보수적 설정. |

**프로젝트 반영**: 변동성 돌파(한국 k 동적), 시초가/돌파 시 유동성 300억 — [00-strategy-registry.md §2.3, §3.2](https://github.com/...).

### 3.4 팩터 투자 (Factor Investing)

| 항목 | 내용 |
|------|------|
| **엣지** | Value·Momentum·Quality·Low Vol 등 공통 팩터 프리미엄. 다중 팩터 조합으로 단일 팩터 의존 완화. |
| **진입** | 팩터 Z-score 또는 분위 합산(예: PEG + Rule of 40 상위 10%); 유니버스·유동성 필터. |
| **청산** | 리밸런싱 주기(월/분기), 팩터 점수 하락, 또는 MDD/손절 규칙. |
| **포지션** | 변동성 역가중, 섹터 집중도 제한(예: 단일 섹터 30% 상한). |
| **기대** | 장기에서 위험 조정 수익 개선; IC/IR 모니터링으로 팩터 퇴화 시 비중 조정. |

**프로젝트 반영**: 퀄리티-성장(PEG, Rule of 40), Factor Zoo(IC/IR·등급), Sector RS — [00-strategy-registry.md §3.2, §6](https://github.com/...).

### 3.5 시장 미세구조 (Market Microstructure)

| 항목 | 내용 |
|------|------|
| **엣지** | 호가·체결·주문 흐름 비균형에서 단기 방향성 추정. 수급 강도(Smart Money), 체결강도. |
| **진입** | 기관/외국인 순매수 2일+ & 체결강도 120%+ 등; RSI·MACD로 방향 확인. |
| **청산** | 수급 반전, -3% Trailing Stop, -5% 손절. |
| **포지션** | 단기·고빈도에 가까우면 1회 리스크 % 엄격, 종목 수 제한. |
| **기대** | 단기 알파; 데이터/지연 비용 민감 → 수집·지연 가정 문서화. |

**프로젝트 반영**: 수급 강도(Smart Money Intensity), 한국 Case A 모멘텀 — [00-strategy-registry.md §3.2](https://github.com/...).

## 4. 검증 및 운영

- **백테스트**: 수수료·슬리피지·PIT·생존자 편향 제거; Walk-Forward·OOS 필수.
- **목표 메트릭**: CAGR≥20~30%, MDD≥-15%, Sharpe≥1.0 (프로젝트 목표와 정렬).
- **거버넌스**: 전략이 더 이상 말이 안 되면 즉시 거래 중단; 버전 스택에 결과·교훈 기록.
- **문서 동기화**: 전략·파라미터·수식 변경 시 [00-strategy-registry.md](../../investment-backend/docs/02-architecture/00-strategy-registry.md) 및 버전 스택 갱신.

## 5. 요약 체크리스트

신규/개선 전략 제안 시 아래를 모두 채운다.

- [ ] **엣지**: 착취 비효율·시간 지평·엣지 원천 명시
- [ ] **진입**: 정량 조건·필터·PIT 준수
- [ ] **청산**: 익절·손절·시간 컷
- [ ] **포지션**: 리스크 기반·캡·변동성 역가중 등
- [ ] **리스크**: MDD·일일 한도·레짐 게이트·Kill Switch
- [ ] **기대**: 정성/정량 기대 엣지·알려진 한계
- [ ] **검증**: 백테스트 가정·Walk-Forward·스트레스 시나리오

이 문서는 **전략 설계 시 단일 참조**로 사용하며, 실제 수식·파라미터·설정값은 전략 레지스트리(00-strategy-registry.md)를 최종 소스로 유지한다.
